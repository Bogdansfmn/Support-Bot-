## 🎯 Описание проекта

**Support Bot** — это микросервис для автоматизации первой линии технической поддержки. Он принимает входящие сообщения от пользователей через вебхуки от платформы **СпросиИИ**, анализирует их с помощью языковой модели (LLM), определяет категорию и приоритет обращения, генерирует персонализированный ответ и при необходимости передаёт запрос оператору с полным контекстом диалога.

### 🚀 Основная идея

Сервис работает как промежуточное звено между системой поддержки (СпросиИИ) и операторами. Он автоматически обрабатывает типовые запросы, генерируя живые, человеческие ответы с помощью LLM, снижая нагрузку на сотрудников. Сложные вопросы передаются специалистам с уже подготовленным контекстом.

### 🤖 Почему LLM?

В отличие от классических rule-based ботов, которые отвечают только по шаблонам, наш бот:
- **Понимает контекст** — учитывает историю диалога
- **Генерирует уникальные ответы** — каждый ответ адаптирован под конкретный запрос
- **Работает на русском языке** — с естественной, вежливой речью
- **Работает локально** — через Ollama, без отправки данных в облако

🤖 Интеграция с LLM (Ollama)
Как работает LLM-классификация
Сообщение отправляется → бот получает текст и контекст

Формируется системный промпт → определяет роль и стиль ответа

Формируется пользовательский промпт → сообщение + контекст + категория

Запрос к Ollama → отправляется через HTTP API

Генерация ответа → LLM создает персонализированный ответ

Возврат результата → ответ отправляется пользователю

Системный промпт (что видит LLM)
text
Ты — профессиональный AI-ассистент службы технической поддержки компании.
Твоя задача — помогать пользователям решать их проблемы быстро и вежливо.

Правила:
1. Отвечай на русском языке, вежливо и профессионально
2. Будь конкретным и давай четкие инструкции
3. Если не знаешь точного ответа, предложи связаться с оператором
4. Структурируй ответ: приветствие, основная информация, завершение
5. Не используй сложный технический жаргон без необходимости
6. Всегда предлагай дальнейшие шаги или уточняющие вопросы

Ты должен быть дружелюбным, но профессиональным.
Пример промпта для пользователя
text
Пользователь обратился в техподдержку со следующей проблемой:

Сообщение: Не могу войти в аккаунт, забыл пароль

Категория проблемы: authorization
Приоритет: high

Пожалуйста, дай развернутый и полезный ответ пользователю.

Когда используется база знаний?
База знаний используется как FALLBACK (резервный вариант):

✅ Если LLM отключена (LLM_ENABLED=False)

✅ Если Ollama не отвечает (ошибка или таймаут)

✅ Если модель не загружена

💾 Хранение истории (Redis)
Как работает хранение истории
Каждое сообщение сохраняется в Redis (или в памяти как fallback)

Ключ: dialog:{user_id}

Значение: список JSON-сообщений

Ограничение: последние N сообщений (настраивается MAX_HISTORY)

Время жизни: 7 дней (настраивается в коде)

Формат хранения
json
{
  "message": "Не могу войти в аккаунт",
  "timestamp": "2026-07-09T10:00:00.000Z",
  "role": "user"
}
Зачем нужен Redis?
Без Redis (в памяти)	С Redis
❌ Потеря истории при перезапуске	✅ История сохраняется
❌ Нет масштабирования	✅ Поддержка нескольких экземпляров
❌ Ограниченный объём	✅ Настраиваемое время жизни

🔧 Примеры использования
1. Отправка запроса через PowerShell
powershell
$body = @{
    event = "message_created"
    id = 123
    content = "Не могу войти в аккаунт, забыл пароль"
    message_type = "incoming"
    private = $false
    sender = @{ id = 456; name = "John Doe" }
    account = @{ id = 1; name = "My Account" }
    conversation = @{ id = 789; status = "open" }
    inbox = @{ id = 10; name = "Support" }
    attachments = @()
} | ConvertTo-Json -Depth 10

Invoke-RestMethod -Uri http://localhost:8000/webhook/ `
    -Method POST `
    -Body $body `
    -ContentType "application/json"
2. Отправка запроса через curl
bash
curl -X POST http://localhost:8000/webhook/ \
  -H "Content-Type: application/json" \
  -d '{
    "event": "message_created",
    "id": 123,
    "content": "Не могу войти в аккаунт",
    "message_type": "incoming",
    "private": false,
    "sender": {"id": 456, "name": "John Doe"},
    "account": {"id": 1, "name": "My Account"},
    "conversation": {"id": 789, "status": "open"},
    "inbox": {"id": 10, "name": "Support"},
    "attachments": []
  }'
3. Получение истории
bash
curl http://localhost:8000/history/456
4. Проверка состояния
bash
curl http://localhost:8000/health
5. Получение статистики
bash
curl http://localhost:8000/stats
